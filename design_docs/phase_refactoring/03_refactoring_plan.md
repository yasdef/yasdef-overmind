# Phase Refactoring: Refactoring Plan (Roundtable)

Status: **agreed roadmap sketch, pre-implementation.** Targets the roundtable design in `02_phase_redesign_and_target_architecture.md`. The earlier consolidation plan is preserved in `design_docs/phase_refactoring_light_version/03_refactoring_plan.md` and serves as the fallback path if the Phase 0 pilot fails.

- **Phase 0 — manual pilot (no tooling needed, can start immediately).** Run one small feature through a hand-moderated roundtable in the experimental workspace: the operator opens one agent session per class (each given only its repo + that repo's `AGENTS.md` + the draft plan + the treaty) and one PO-agent session; questions and answers are relayed by hand; the transcript is kept as a plain file. Measure: round count to convergence, discoveries made vs. what the old pipeline found on a comparable feature, wall time, and questionnaire coverage holes. Also test D3 cheaply: distill one step's seat answers into a step-design artifact and check it against the worker's existing design readiness gate (`check_design_readiness.py`). Exit criterion: go/no-go on the roundtable direction plus a battle-tested questionnaire v1. If no-go → fall back to the light version.
- **Phase 1 — settle pending decisions.** From `02_phase_redesign_and_target_architecture.md ## 8. Settled Decisions, Pending Decisions, Deferred Topics`: questionnaire template (informed by the pilot), transcript format including the machine-readable trace the gates consume and session resumability, living-inventory bootstrap and update timing, seat context recipe.
- **Phase 2 — specify.** Target step definitions for F1–F3 (YAML as canonical source, diagram derived), artifact templates (transcript, contract-delta landing rules), and gate specs (plan readiness, dialogue completeness, mirror check, `plan_approved_by_operator`).
- **Phase 3 — implement in the TS port.** Seat runner and gates land as skills + `asdlc-coordinator` per `design_docs/to_skills_migration/`; do not patch the shell+md pipeline. Note: no yasdef worker changes are required for planning itself; the lazy sign-off uses the worker's existing design phase and `raised_to_coordinator` disposition. D3 needs one small, independently shippable worker change: if a valid step-design artifact already exists for the selected step (existing design readiness gate decides validity), skip the design phase and start from step-plan.
- **Phase 4 — validate.** Parallel-run one feature on an experimental project against the current pipeline; compare plan quality, discoveries, wall time, and gate behavior.
- **Phase 5 — cut over.** Retire old feature-phase steps 6–8.4 and their templates; update `overmind/README.md` and the sequence diagram (generated/derived from YAML).
