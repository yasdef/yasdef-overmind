## What Changes

- Add mandatory `asdlc-coordinator/workspace` and `asdlc-coordinator/sequencing` package subpath exports, then add a minimal read-only VS Code dashboard in `packages/vscode-extension` (activation plus a contributed read-only view) that imports only those subpaths, obtains the extension `FeatureSummary` data contract (`readiness`, `completedSteps`, `totalSteps`, `missingArtifacts`) by reusing the existing `sequencing/toFeatureSummary()` export rather than re-mapping `ProgressReport`, and couples to zero orchestrator-CLI surface — proving reuse claim 5 in `04_migration_plan.md ## Definition of done (whole effort)`.
- Remove now-orphaned `common_libs` helpers and their staging entries after auditing that no functional production consumer (an active runtime/setup/update script that sources, executes, or otherwise invokes them) still depends on them — judging liveness without counting the helper's own staging entry or dedicated test — and add an update-mode cleanup — an explicit `OBSOLETE_STAGED_COMMAND_LIB_FILES` tombstone sweep that deletes only named stale libs (no blanket managed-manifest deletion, since the staged `common_libs` directory preserves unmanaged operator content and is not package-owned) — so removed helpers are deleted from already-deployed workspaces, given that `stage_command_libs()` only copies and never removes stale files. Retain the `OBSOLETE_STAGED_COMMAND_FILES` tombstones, which still delete legacy scripts from workspaces upgrading directly from older releases.
- Sweep the versioned operator docs — root `README.md` (the file the plan's `overmind/README.md` reference resolves to; no `overmind/README.md` exists) and `QUICKRUN.md` — and repository references for dead mentions of deleted shell scripts, stale entrypoints, and superseded workflows.
- Revise the extension design docs (`design_docs/overmind_vscode_extention/requirements_ears.md`, `technical_requirements.md`, `implementation_plan.md`) to replace every `.commands/*.sh` launching surface — Requirement 7 clauses, Requirement 9 verification, the Run Scanner / Create Feature / Continue E2E actions, and the script allow-list — with the shipped `overmind` verbs and in-process core per `03_target_architecture.md ### Supersession of the extension design docs`, remove scanner freshness/stale-state caveats in favor of fresh in-process recomputation, explicitly postpone the unmapped Create Project terminal action until a shared coordinator primitive or shipped verb exists, and remove the supersession banners. This revision lands **before** the dashboard is implemented, so the extension is built against the corrected plan (per the plan's "Extension implementation must not restart from the old plan before this revision lands").
- Fold the executed behavior-level sweep result into `02_responsibility_translation_map.md ## Full parity gate` and mark it closed there, and retire the "detection only" caveat on rows 18–20 now that Slice 4 landed their execution tests — auditing that every row is marked owned-or-retired.
